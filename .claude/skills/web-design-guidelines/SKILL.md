# Web Design Guidelines

Use this skill for UI review, audit, or refinement work. Focus on practical web design quality rather than personal taste.

## Principles

- Prioritize clarity over decoration.
- Build strong visual hierarchy with spacing, size, weight, and contrast.
- Make interactions obvious and responsive.
- Design for accessibility from the start.
- Adapt layouts to different screen sizes and input methods.
- Use motion to support comprehension, not to distract.
- Keep copy concise, specific, and action-oriented.

## Review Areas

### Layout and Hierarchy

- Is the primary action obvious?
- Can the page be scanned quickly?
- Are groups separated clearly through spacing and alignment?
- Are cards, borders, and containers used intentionally instead of by default?

### Typography

- Is the type scale clear and consistent?
- Are long lines constrained for readability?
- Are secondary labels visually subordinate without becoming illegible?

### Color and Contrast

- Are semantic colors used consistently?
- Does all important text meet readable contrast?
- Are status colors supported by labels or icons instead of color alone?

### Components and States

- Do buttons, forms, and navigation controls have clear states?
- Are focus styles visible?
- Are loading, empty, success, and error states handled well?

### Responsive Behavior

- Does the layout adapt rather than just shrink?
- Are touch targets large enough?
- Are hover-only affordances avoided on touch-first surfaces?

### UX Writing

- Are button labels specific?
- Are error messages actionable?
- Do empty states explain value and next steps?

## Output Style

When reviewing:

- Call out the highest-impact issues first.
- Explain why each issue matters.
- Suggest concrete, realistic improvements.

When refining:

- Make changes that clearly improve usability or perceived quality.
- Prefer a small set of high-leverage fixes over a broad noisy pass.
